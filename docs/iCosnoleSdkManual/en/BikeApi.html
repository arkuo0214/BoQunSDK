Bike

Bike API

BoQunBike

Parameter Description
BoQunBike.TYPE_SPORT_STATUS Sport status. Use queryMachineInfo to query and then call back onResponseSportStatus
BoQunBike.TYPE_LOAD_LEVEL Resistance level, query using queryMachineInfo and then call back onResponseLoadLevel
Function Description
BoQunBike.init(OnBikeListener listener) Initializes the serial port and returns a Boolean value indicating whether the serial port was successfully initialized.
BoQunBike.queryConfigInfo() Query the bicycle configuration information resistance, lifting range, status, etc., callback onResponseConfigInfo, read failure callback onError
BoQunBike.queryMachineInfo( int type) Query bicycle information, parameter type
BoQunBike.start() Start exercising, call back onResponseSportStatus after successful start, and start transmitting data onResponseSportData
BoQunBike.pause() Pause the sport. After the sport is successfully paused, the onResponseSportStatus callback is called. During the pause state, the sport data will not be transmitted.
BoQunBike.stop() Stop exercising, call back onResponseSportStatus
BoQunBike.setLoadLevel( int load ) Control the resistance level of the bicycle, callback onResponseLoadLevel, the larger the load value, the greater the resistance, the load has a range limit, the load range parameter reads BikeConfigInfo
BoQunBike.setInclineLevel( int incline ) Control the bicycle incline level, callback onResponseInclineLevel, the larger the incline value, the higher the incline. The incline has a range limit, and the incline range parameter reads BikeConfigInfo
BoQunBike.setFanLevel( int level) Control the fan wind speed level, callback onResponseFanLevel, the wind speed level supports 0-3 levels, if the machine does not have a fan, it does not support adjustment, get whether the machine has a fan status BikeConfigInfo
BoQunBike.setWorkMode( int mode) Set the bicycle lower computer working mode BikeWorkMode
BoQunBike.setFTMSEnabled( boolean enable) Enable or disable FMTS. It must be used with a lower control board with FMTS function. Callback BikeStatusCallbacks.FTMSStateCallback
BoQunBike.setSerialPortEnabled( boolean enable) Enable and disable the serial port. The default is enabled. This method is provided to facilitate handling special situations. For example, multiple apps reading the serial port at the same time will cause mutual interference. This method can be used to avoid tips: It will not really close the serial port, but it will definitely not read the serial port data
BoQunBike.setSportData(long time, float speed, float distance, int calories, int rpm, int pulse, int watt) When the APP and the electronic watch display exercise data synchronously, it is usually necessary to send the exercise data to the lower computer. If there is no requirement, please ignore it

- time unit: seconds, range (0-65535)
- speed unit: kilometers per hour, range (0-65535)
- distance unit: km, range (0-65535)
- calories unit: kcal, range (0-65535)
- rpm range (0-65535)
- pulse range (0-65535)
- watt range (0-65535)
BoQunBike.setSportParams(@BikeSportParams int code, int value) Set sport parameters. Ignore if not required
See BikeSportParams for parameter details
BoQunBike.setBikeListener(OnBikeListener listener) Bicycle Listener (deprecated)
BoQunBike.registerStateListener(BikeStatusCallback callback) Register status callback listener
BoQunBike.unregisterStateListener(BikeStatusCallback callback) Unregister status callback listener
BoQunBike.unregisterAllStateListener() Unregister all status callback listeners
BoQunBike.destroy() Destroy the instance and release resources

BikeWorkMode

Parameter Description
BikeWorkMode.NORMAL This state should be set when the TFT screen is on and enters the working state
BikeWorkMode.SLEEP This state should be set when the TFT screen is off and enters sleep mode
BikeWorkMode.SHUT_DOWN This state should be set when TFT is turned off to release resources

BikeStatusCallback

Function Description
void onBikeSportStateChange(int status) Bicycle status change callback

BikeStatusCallbacks

This class mainly implements status callback. It currently supports monitoring FTMS status. Other functions will be implemented later if needed, or you can inherit the BikeStatusCallback interface to implement custom status callback.

Currently implements FMTS switch status callback

BoQunBike.registerStateListener(new BikeStatusCallbacks.FTMSStateCallback() {
            @Override
            public void onFTMSStateChange(boolean enable) {
                 //Enable and disable status
            }
        });

Remember to unregister

BoQunBike.unregisterStateListener(xxxx);

//Or, look at your personal application scenarios and needs
BoQunBike.unregisterAllStateListener();

OnBikeListener

Function Description
void onResponseConfigInfo(BikeConfigInfo info) Reply to the control configuration information
void onResponseSportStatus(@BikeSportState int status) Respond to sport status, parameter: status
void onResponseSportData(int stepCount, int rpm, int heartRate) Reply to exercise data
stepCount: number of steps (accumulated value)

rpm: revolutions per minute

heartRate: heart rate when holding the hand
void onResponseLoadLevel(int load) Restore the resistance level of the lower control
void onResponseInclineLevel(int incline) Resume control and ascend level
void onResponseFanLevel(int level) Reset fan speed level
void onExternalKeyEvent(@BikeKeyCode int keyCode) Pressing an external key on the panel (some UP and DOWN buttons can be long pressed) triggers a callback and replies BikeKeyCode
void onError(int errorCode) Error callback occurs, parameters see ErrorCode, treadmill down-control error code will also be called back, but the treadmill error code depends on the error code provided by the docking motor manufacturer.

BikeSportState

Parameter Description
BikeSportState.STOP Stop state
BikeSportState.START Starting state
BikeSportState.PAUSE Pause state

BoQunBikeFactory

Function Description
BoQunBikeFactory.init() Instantiating the bicycle factory
BoQunBikeFactory.queryLoadMotorStroke( int load) Query the resistance motor stroke of the specified resistance level and call back onResponseLoadMotorStoke
BoQunBikeFactory.queryInclineMotorStroke( int incline) Query the incline motor stroke of the specified incline level, and call back onResponseInclineMotorStroke
BoQunBikeFactory.setWheelDiameter( int wheelDiameter) Set the wheel diameter
BoQunBikeFactory.setLoadMotorStroke( int load, int adc) Set the resistance motor travel ADC value for the specified resistance level
BoQunBikeFactory.setInclineMotorStroke( int incline, int adc) Set the ADC value of the lifting motor stroke for the specified lifting level
BoQunBikeFactory.startInclineAutoCorrection() Start automatic correction of the lifting motor, callback onResponseInclineAutoCorrection
BoQunBikeFactory.startInclineAutoCorrection(@AutoCorrectionDirection int direction) Start automatic correction of the lifting motor, callback onResponseInclineAutoCorrection

direction Correction direction

- AutoCorrectionDirection.AUTO The default direction is automatic
AutoCorrectionDirection.UP AutoCorrectionDirection.DOWN
BoQunBikeFactory.stopInclineAutoCorrection() Stop the automatic correction of the lifting motor and call back onResponseInclineAutoCorrection
BoQunBikeFactory.setOnBikeFactoryListener( OnBikeFactoryListener listener) Bicycle factory mode listener
BoQunBikeFactory.destroy() Destroy the instance and release resources

OnBikeFactoryListener

Function Description
void onResponseLoadMotorStoke(int load, int adc) Resume the motor stroke ADC for the specified resistance level
void onResponseInclineMotorStroke(int incline, int adc) Restore the motor stroke ADC of the specified ascension level
void onResponseInclineAutoCorrection(@AutoCorrectionStateint state, int inclineADC) Respond to the auto-correction status and ADC value, parameter: state

AutoCorrectionState

inclineADC: value range 0-255. ADC will keep changing during the calibration of the lifting motor. Normally, it will change from small to large, and from large to small is a complete calibration.

Parameter Description
AutoCorrectionState.CORRECTION_START Lifting motor calibration starts
AutoCorrectionState.CORRECTION_ING Lifting motor calibration
AutoCorrectionState.CORRECTION_STOP Lifting motor calibration stop

getMaxInclineLevel() int Get the maximum ascension level

hasFan() boolean Is the machine equipped with a fan
hasFTMS() boolean Is the machine equipped with FMTS function
